Automated cell filtering for single-cell RNA sequencing data

dropkick is a quality control and cell identification tool for single-cellRNA sequencing (scRNA-seq) data that works seamlessly with scanpyâ€™s AnnData objects in Python. This tool accepts input files in compressed .h5ad or flat format (.csv or .tsv) with barcodes as rowsand genes as columns. Row and column headers should be present in flat files. Outputs will be written to a .csv table containing final dropkick scores and labels, as well as a compressed .h5ad file with original data for scanpy processing.

Background

Genome Research Article


A major challenge for droplet-based single-cell sequencing technologies is distinguishing true cells from uninformative barcodes in datasets with disparate library sizes confounded by high technical noise (i.e. batch-specific ambient RNA). We present dropkick, a fully automated software tool for quality control and filtering of single-cell RNA sequencing (scRNA-seq) data with a focus on excluding ambient barcodes and recovering real cells bordering the quality threshold. By automatically determining dataset-specific training labels based on predictive global heuristics, dropkick learns a gene-based representation of real cells and ambient noise, calculating a cell probability score for each barcode. Using simulated and real-world scRNA-seq data, we benchmarked dropkick against conventional thresholding approaches and EmptyDrops, a popular computational method, demonstrating greater recovery of rare cell types and exclusion of empty droplets and noisy, uninformative barcodes. We show for both low and high-background datasets that dropkick's weakly supervised model reliably learns which genes are enriched in ambient barcodes and draws a multidimensional boundary that is more robust to dataset-specific variation than existing filtering approaches. dropkick provides a fast, automated tool for reproducible cell identification from scRNA-seq data that is critical to downstream analysis and compatible with popular single-cell Python packages.

Full documentation is available at KenLauLab.github.io/dropkick.

Example

Sample Results for 4k Human T Cell Data


QC Plot

This human T cell dataset exhibits a high-quality, low-background scRNA-seq sample. The solid black curve in the log-rank plot on the left describes the total UMI counts per ranked barcode in the dataset. The green points show the corresponding number of genes detected in each barcode, while the blue and red points calculate the percentage of ambient and mitochondrial expression in each barcode as an indicator of quality.

The gene dropout plot on the right shows the percentage of barcodes in which each ranked gene is undetected. The bottom genes by dropout rate are flagged as "ambient" by dropkick.



Score Plot

The dropkick score plot displays these barcodes in a scatter plot of ambient percentage (as calculated in the dropkick QC plot above) versus total genes detected for each cell. The automated thresholds on the latter heuristic are overlayed to show the initial training labels given to the dropkick model.



Coefficient Plot

The dropkick coefficient plot displays final model coefficient values and cross validation (CV) scores for the tested regularization values of lambda. Top and bottom three genes by coefficient are notated for reference.



dropkick Outputs

dropkick returns an AnnData object with the following attributes. You may also download a table containing dropkick scores and labels for each barcode as a .csv file (see example below).

  • .obs['dropkick_score']
    • Probability of being a real cell
    • Values closer to 0 indicate empty droplets with high ambient RNA content
  • .obs['dropkick_label']
    • Binary label generated by using cutoff of 0.5 on 'dropkick_score'
    • dropkick's best estimate of a "good" dataset
    • User is encouraged to adjust the cutoff according to whether they are more concerned with false negatives or false positives
  • .obs[<metrics>]
    • Metrics used for thresholding (metrics parameter, default ['arcsinh_n_genes_by_counts'], are kept
  • .var['dropkick_coef']
    • Logistic regression coefficient values for each highly variable gene (HVG) determined by the dropkick pipeline
  • .uns['dropkick_thresholds']
    • Dictionary of global metrics thresholded by dropkick with the resulting threshold values and directions for good training cells
  • .uns['dropkick_args']
    • Dictionary of all arguments used to run the pipeline
